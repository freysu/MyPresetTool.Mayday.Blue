预设代码生成器-使用指南
  1. 一、上传音频（必填）
    • audio_file上传音频文件
      在对应区域点击“选择文件”，从您设备选一首任意的音频（支持MP3、WAV、FLAC、AAC等格式）。
      此音频的节奏、旋律等特征将用于生成预设代码文本，例如节奏影响颜色变化速度，旋律与颜色亮度、饱和度相关。
    • description上传歌词文件（可选）
      若有对应.lrc格式的歌词文件，可在对应区域上传。
      这能助系统精准理解音乐情感，使生成的颜色序列更贴合歌曲意境。例如，欢快的歌词对应明亮鲜艳的颜色，抒情部分对应柔和温暖的颜色。
  2. 二、颜色主题配置（必填）
    • add_circle1.在Base Color下拉菜单中挑选能代表该歌曲整体感觉的基础颜色。
    • add_circle2.配置动态颜色范围，确保各范围总和为100%
    • save3.完成颜色主题配置后，请点击保存!
      file_upload若使用已有配置文件，可以点击导入并选择相应的.json文件导入。
      file_download若想在其他歌曲中复用或分享创意，可以点击导出来保存.json格式的文件。
    • palette可用颜色列表
      • red：红色
      • ora：橙色
      • yel：黄色
      • sky：天蓝色
      • blu：蓝色
      • pur：紫色
    • palette动态颜色范围解释：
      • Low Colors:对应音频柔和低频部分（前奏、间奏、节奏慢段落）
      • Mid Colors:对应中等强度频率部分（主歌、节奏适中段落）
      • High Colors:对应音频强烈高频部分（副歌高潮、节奏强烈段落）
      • Accent Colors:对应音频的关键处（旋律转折、情感爆发点）
  3. 三、生成并获取颜色序列
    • play_arrow生成预设代码文本
      点击生成，程序会依据上传的音频及配置的颜色主题在输出结果区域呈现生成的预设代码。
    • content_copy复制与下载结果
      可通过点击复制粘贴到预设代码可视化预设代码可视化这个选项卡里预览效果或点击下载即可保存为.json文本文件，方便离线查看、编辑或分享。
  4. 四、示例颜色主题配置JSON文件:
    • 1.示例JSON中low设置为100%白色。
      2.示例JSON中Mid配置为30%黄色（yel）和70%橙色（ora）。颜色依此比例在黄橙之间混合变化，反映音乐的情感节奏，黄色欢快明亮，橙色增强活力，与音频此部分呼应。
      3.示例JSON中High设置为30%橙色（ora）和70%红色（red）。红色表示强烈激情，在高潮部分高比例呈现，与橙色混合展现音乐高潮的热烈氛围，随音频变化在橙红之间动态过渡。
      4.示例JSON中Accent中与high范围相同（30%橙色ora和70%红色red），在关键处（旋律转折、情感爆发点）呈现。
    •

                      "low": [{ "color": "whi", "per": "100%" }],
                      "mid": [
                        { "color": "yel", "per": "30%" },

                      "accent": [
                        { "color": "ora", "per": "30%" },
                        { "color": "red", "per": "70%" }
                      ],
                      "base": "ora"
                    }
                  }

预设代码可视化工具-使用指南
  1. 一、上传音频（必填）
    • audio_file 上传音频文件
      在对应区域点击“选择文件”，从您设备选一首任意的音频（支持MP3、WAV、FLAC、AAC等格式）。
      此音频的节奏、旋律等特征将用于生成预设代码文本，例如节奏影响颜色变化速度，旋律与颜色亮度、饱和度相关。
  2. 二、上传预设代码文件或输入预设代码文本（可选）
    • file_upload 上传预设代码文件
      若有对应的预设代码文件，可以在对应区域点击“选择文件”进行上传。
      预设代码文件可以帮助系统更好地理解音乐的情感和节奏，从而生成更符合预期的颜色序列。
    • edit 手动输入预设代码
      如果没有预设代码文件，可以在预设代码输入区手动输入预设代码。
      预设代码格式为：时间(ms), 颜色。例如：0, pin2；500, pin2；1000, sky3；1500, pin4。
  3. 三、查看颜色预览和音频可视化效果
    • palette 颜色预览区
      在颜色预览区可以看到根据预设代码生成的颜色变化效果。
      颜色预览区会实时显示当前时间点的颜色，并显示颜色变化的时间轴。
    • music_note 音频可视化
      在音频可视化区域可以选择不同的可视化模式（频谱图或波形图），以直观地看到音频的变化。
      音频可视化可以帮助用户更好地理解音频的节奏和旋律，从而调整预设代码以达到更好的效果。
  4. 四、使用控制按钮或快捷键操作动画
      提示: [Windows :Ctrl, OS X: Command!]
      按下"Ctrl/Cmd + ."可以查看快捷键说明！在编辑文本时，快捷键会暂时禁用。切到其他选项卡会暂时禁用。
    • play_arrow 开始 (Ctrl/Cmd + Enter)
      点击“开始”按钮，动画将按照预设代码的时间和颜色变化开始播放。
      动画开始后，可以在颜色预览区和音频可视化区域看到效果。
    • pause 暂停 (Space)
      点击“暂停”按钮，动画将暂时停止播放。
      暂停后，可以通过点击“继续”按钮恢复播放。
    • play_arrow 继续 (Space)
      点击“继续”按钮，动画将从暂停的位置继续播放。
      继续播放后，可以在颜色预览区和音频可视化区域看到效果。
    • stop 停止 (Esc 或 Ctrl/Cmd + S)
      点击“停止”按钮，动画将完全停止播放。
      停止后，可以通过点击“开始”按钮重新开始播放。
    • replay 重新开始 (Ctrl/Cmd + R)
      点击“重新开始”按钮，动画将从头开始播放。
      重新开始后，可以在颜色预览区和音频可视化区域看到效果。
    • clear 清除 (Ctrl/Cmd + Backspace)
      点击“清除”按钮，将清除所有预设代码和动画效果。
      清除后，可以重新上传音频和预设代码文件，或手动输入新的预设代码。
